# คู่มือการใช้งาน WebAutomation Studio

ฉบับย่อแบบเป็นทางการ

คู่มือนี้จัดทำขึ้นเพื่ออธิบายการใช้งานโปรแกรม **WebAutomation Studio** สำหรับเปิดหน้าเว็บ กรอกข้อมูล คลิกปุ่ม และรันงานอัตโนมัติผ่าน Selenium โดยรองรับทั้ง **Builder Mode** และ **Python Mode** รวมถึงการใช้ข้อมูลจาก **JSON/CSV** เพื่อวนทำงานหลายรายการ 

---

## 1. ภาพรวมของโปรแกรม

โปรแกรมประกอบด้วย 3 ส่วนหลัก

### 1.1 Custom Chrome Setup

ใช้กำหนดค่าการเปิดเบราว์เซอร์ ได้แก่

* Profile Folder
* Chrome.exe
* Chromedriver.exe

เมื่อกำหนดครบแล้วจึงสามารถเริ่ม Browser ได้ หากข้อมูลไม่ครบ โปรแกรมจะไม่อนุญาตให้เริ่มทำงาน 

### 1.2 Automation Studio

เป็นพื้นที่หลักสำหรับสร้างลำดับคำสั่งอัตโนมัติ โดยมี 2 โหมด

* **Builder Mode** สำหรับใช้งานแบบบล็อกคำสั่ง
* **Python Mode** สำหรับเขียนโค้ดควบคุมเอง

### 1.3 About

แสดงข้อมูลสรุปของโปรแกรม

## 2. ข้อควรระวังก่อนเริ่มใช้งาน

### 2.1 ห้ามปิดแท็บแรกที่โปรแกรมเปิดขึ้นมา

หลังจากกด **เริ่ม Browser** โปรแกรมจะเปิด Chrome และเข้าสู่หน้าเริ่มต้นให้โดยอัตโนมัติ หากผู้ใช้ปิดแท็บแรกนั้นเอง อาจทำให้ลำดับการทำงานภายหลังเกิดข้อผิดพลาดได้ เนื่องจากการควบคุมของ Selenium ยังอ้างอิงกับหน้าต่างและแท็บที่ถูกเปิดไว้ตั้งแต่ต้น

**คำแนะนำ:**
ควรปล่อยแท็บแรกไว้ตลอดการทำงาน และให้โปรแกรมเป็นผู้เปลี่ยน URL ผ่านคำสั่ง `Goto URL` หรือคำสั่งใน Python Mode แทน

### 2.2 ต้องใช้ Chrome และ Chromedriver ที่เข้ากัน

หากเวอร์ชันไม่สอดคล้องกัน อาจเปิด Browser ไม่ได้หรือเกิดข้อผิดพลาดระหว่างทำงาน

### 2.3 ไม่ควรเปิดใช้ Profile เดียวกันซ้อนกันหลายหน้าต่าง

อาจทำให้ Chrome เปิดไม่ขึ้น หรือ session เกิดการชนกัน

## 3. หลักการทำงานพื้นฐาน

การสั่งงานเว็บอัตโนมัติจำเป็นต้องระบุว่าโปรแกรมจะไปจับ **element** ตัวใดบนหน้าเว็บ เช่น

* ช่องกรอกข้อความ
* ปุ่มส่ง
* ช่องค้นหา
* กล่องข้อความยาว

โปรแกรมจึงรองรับการค้นหา element หลายรูปแบบ เช่น

* id
* name
* css
* xpath
* class_name
* tag_name 

---

## 4. การ Inspect HTML เบื้องต้น

การ Inspect ใช้สำหรับดูโครงสร้าง HTML ของหน้าเว็บ เพื่อค้นหาว่าควรอ้างอิง element ด้วยวิธีใด

### วิธีเปิด Inspect

ใน Chrome ให้

* คลิกขวาที่ช่องหรือปุ่มที่ต้องการ
* เลือก **Inspect**
* หรือกด `F12`

### สิ่งที่ควรมองหา

ให้ตรวจสอบข้อมูลต่อไปนี้

* `id`
* `name`
* `class`
* `placeholder`
* `aria-label`
* ข้อความที่แสดงบนปุ่มหรือ label
* โครงสร้างพ่อแม่ลูกของ element

ตัวอย่าง

```html
<label for="email">อีเมล</label>
<input id="email" name="email" type="text">
```

จากตัวอย่างนี้ ช่องกรอกสามารถอ้างอิงได้ด้วย

* `id = email`
* `name = email`
* XPath ที่อ้างอิง `input`

---

## 5. ความหมายของ XPath

XPath คือรูปแบบการระบุตำแหน่ง element ภายในโครงสร้าง HTML โดยเหมาะสำหรับกรณีที่ต้องการอ้างอิงจากข้อความหรือโครงสร้าง

ตัวอย่างพื้นฐาน

### หา input จาก id

```xpath
//*[@id='email']
```

### หา input จาก name

```xpath
//input[@name='email']
```

### หาปุ่มที่มีข้อความว่า “ส่ง”

```xpath
//button[normalize-space()='ส่ง']
```

### หาข้อความ “ส่ง” ที่อยู่ใน span แล้วไต่ขึ้นไปยังปุ่ม

```xpath
//span[normalize-space()='ส่ง']/ancestor::*[@role='button']
```

รูปแบบสุดท้ายมีประโยชน์มาก เพราะหลายเว็บไซต์แสดงข้อความอยู่ใน `span` แต่ตัวที่คลิกจริงอาจเป็น `button` หรือ `div`

---

## 6. ความหมายของ CSS Selector

CSS Selector เหมาะกับการอ้างอิง element จาก id, class หรือ attribute โดยมีลักษณะกระชับกว่าบางกรณี

ตัวอย่าง

```css
#email
input[name="email"]
button.submit-btn
```

### หลักการเลือกใช้งาน

* หากมี `id` ชัดเจน ควรใช้ `id`
* หากมี `name` ชัดเจน ควรใช้ `name`
* หากต้องอ้างอิงจากข้อความบนหน้าจอ ควรใช้ XPath

---

## 7. แนวทางเลือก selector ที่เหมาะสม

ลำดับความเสถียรโดยทั่วไป

1. `id`
2. `name`
3. `css`
4. `xpath`
5. `class_name`
6. `tag_name`

### หมายเหตุสำคัญ

`class_name` ไม่เหมาะกับ element ที่มี class ซ้ำกันจำนวนมาก และไม่ควรใส่ class หลายชื่อรวมกันเป็นสตริงเดียว

ตัวอย่างเช่น

```html
<span class="NPEfkd RveJvd snByac">ส่ง</span>
```

กรณีนี้ไม่ควรใช้

```text
class_name = "NPEfkd RveJvd snByac"
```

แต่ควรใช้เพียง class เดียว หรือเปลี่ยนไปใช้ CSS/XPath แทน

---

## 8. การใช้งาน Builder Mode

Builder Mode รองรับบล็อกคำสั่งหลักดังนี้ 

### 8.1 Goto URL

ใช้เปิดหน้าเว็บตาม URL ที่กำหนด

**ค่าที่ต้องระบุ**

* URL
* Wait after load (วินาที)

**คำแนะนำ**
ควรใช้เป็นบล็อกแรกของงานส่วนใหญ่ เพื่อให้โปรแกรมไปยังหน้าที่ต้องการก่อนเริ่มกรอกหรือคลิก

---

### 8.2 Fill by Label

ใช้กรอกข้อมูลโดยอาศัยข้อความของ label หรือข้อความคำถามบนฟอร์ม

โปรแกรมจะพยายามหา field ที่เกี่ยวข้องจากหลายรูปแบบ เช่น

* label ที่ผูกกับ `for`
* input/textarea ถัดจาก label
* placeholder
* aria-label
* name 

**ค่าที่ต้องระบุ**

* Label text
* Value
* Field type
* Index
* Timeout

**คำแนะนำ**

* ใช้ `auto` เมื่อไม่แน่ใจว่าเป็น input หรือ textarea
* ใช้ `index` เมื่อมี field ชื่อซ้ำกันหลายช่อง
* `index` เริ่มนับจาก 0

---

### 8.3 Fill by Selector

ใช้กรอกข้อมูลโดยอ้างอิง selector โดยตรง

**ค่าที่ต้องระบุ**

* Selector type
* Selector
* Value
* Timeout

เหมาะสำหรับกรณีที่ Inspect แล้วพบ `id`, `name` หรือ XPath ที่ชัดเจน

---

### 8.4 Click by Text

ใช้คลิก element จากข้อความที่แสดงอยู่บนหน้าจอ เช่น

* ส่ง
* ถัดไป
* ยืนยัน

โปรแกรมจะพยายามค้นหาปุ่มหรือ element ที่เกี่ยวข้องกับข้อความดังกล่าว 

**ข้อสังเกต**
หากข้อความอยู่ใน `span` แต่ตัวที่คลิกจริงคือ `button` หรือ `div` บางครั้งอาจต้องเปลี่ยนไปใช้ Click by Selector แทน

---

### 8.5 Click by Selector

ใช้คลิก element โดยอ้างอิง selector ตรง ๆ

เหมาะสำหรับกรณีที่ Click by Text ไม่สามารถระบุตำแหน่งได้ถูกต้อง

---

### 8.6 Wait

ใช้หน่วงเวลาเป็นวินาที

เหมาะสำหรับ

* รอหน้าเปลี่ยน
* รอ popup
* รอผลลัพธ์หลังคลิกส่ง

---

### 8.7 Press Key

ใช้ส่งคำสั่งคีย์บอร์ดไปยัง element เช่น

* ENTER
* TAB
* ESCAPE
* SPACE 

---

## 9. การใช้ข้อมูลหลายรายการด้วย Dataset

โปรแกรมรองรับข้อมูลจาก

* CSV
* JSON
* การพิมพ์ JSON เองในช่อง Dataset / Rows 

ตัวอย่างข้อมูล

```json
[
  {"ชื่อ":"แบม","อีเมล":"bam@example.com"},
  {"ชื่อ":"เมย์","อีเมล":"may@example.com"}
]
```

เมื่อต้องการดึงค่าจาก dataset ให้ใช้รูปแบบ

```text
{{ชื่อ}}
{{อีเมล}}
```

โปรแกรมจะนำค่าจากแต่ละแถวมาแทนที่โดยอัตโนมัติระหว่างการวนทำงาน 

---

## 10. การใช้งาน Python Mode

Python Mode เหมาะสำหรับกรณีที่ลำดับงานมีเงื่อนไขซับซ้อน หรือ Builder Mode ไม่เพียงพอ

ในโหมดนี้ โปรแกรมเตรียมตัวช่วยไว้ให้แล้ว เช่น

* `driver`
* `rows`
* `row`
* `goto()`
* `fill_by_label()`
* `fill_selector()`
* `click_text()`
* `click_selector()`
* `press_key()`
* `wait()`
* `log()` 

เหมาะสำหรับงานที่ต้องการ

* เขียนเงื่อนไขเอง
* วนลูปแบบกำหนดเอง
* แก้ปัญหาเว็บที่มีโครงสร้างซับซ้อน
* จัดการกรณีพิเศษที่บล็อกสำเร็จรูปไม่รองรับ

---

## 11. Speed Mode

โปรแกรมมีโหมดความเร็ว 4 ระดับ

* safe
* normal
* fast
* turbo 

### แนวทางเลือกใช้

* **safe** สำหรับเว็บช้า หรือมี animation มาก
* **normal** สำหรับงานทั่วไป
* **fast** สำหรับเว็บที่เสถียร
* **turbo** สำหรับกรณีที่มั่นใจมากว่าเว็บตอบสนองรวดเร็ว

หากเลือกเร็วเกินไป อาจเกิดปัญหา element ยังไม่พร้อมแต่โปรแกรมเริ่มทำงานแล้ว

---

## 12. Import และ Export Builder JSON

โปรแกรมสามารถ

* **Export Builder JSON** เพื่อบันทึกลำดับบล็อกคำสั่ง
* **Import Builder JSON** เพื่อนำลำดับเดิมกลับมาใช้งานอีกครั้ง 

เหมาะสำหรับ

* การเก็บ template งาน
* การใช้งานซ้ำ
* การย้ายชุดคำสั่งระหว่างงาน

---

## 13. แนวทางแก้ปัญหาเบื้องต้น

### กรณีหา field ไม่เจอ

ให้ตรวจสอบว่า

* label ที่ใช้ตรงกับข้อความบนหน้าเว็บจริงหรือไม่
* element เป็น `input` หรือ `textarea`
* มีข้อความซ้ำหลายจุดหรือไม่
* ควรใช้ selector แทน label หรือไม่

### กรณีคลิกปุ่มไม่ได้

ให้ตรวจสอบว่า

* ตัวที่กดจริงเป็น `button`, `div`, หรือ `span`
* มี popup หรือ overlay บังอยู่หรือไม่
* ควรใช้ XPath ที่ไต่ไปยัง parent element หรือไม่

### กรณีเกิด Timeout

สาเหตุหลักมักเกิดจาก

* selector ไม่ถูกต้อง
* หน้าเว็บยังโหลดไม่เสร็จ
* element ยังไม่ clickable
* ผู้ใช้ปิดแท็บที่เปิดไว้ตอนเริ่มต้น

---

## 14. ข้อแนะนำในการใช้งานจริง

1. เริ่มจากทดสอบทีละบล็อก
2. ใช้ `id` หรือ `name` ก่อน หากมี
3. ใช้ XPath เมื่อจำเป็นต้องอ้างอิงจากข้อความ
4. หลีกเลี่ยง XPath ที่ยาวและอิงตำแหน่งล้วน
5. อย่าปิดแท็บแรกที่โปรแกรมเปิดขึ้นมา
6. บันทึก flow ที่ใช้งานได้แล้วด้วย Export JSON
7. หากเว็บมีพฤติกรรมซับซ้อน ให้เปลี่ยนไปใช้ Python Mode

---

## 15. สรุป

WebAutomation Studio เป็นเครื่องมือสำหรับสร้างงานอัตโนมัติบนหน้าเว็บโดยใช้ Selenium ผ่านหน้าต่าง GUI ผู้ใช้สามารถเลือกได้ระหว่างการประกอบบล็อกคำสั่งแบบ Builder หรือเขียนโค้ดเองใน Python Mode พร้อมรองรับการทำงานแบบหลายแถวผ่าน dataset จาก JSON/CSV ได้ในตัว 

เพื่อให้ใช้งานได้อย่างเสถียร ควรยึดหลักดังนี้

* ตั้งค่า Chrome ให้ถูกต้อง
* Inspect HTML ก่อนสร้างคำสั่ง
* เลือก selector ที่เสถียรที่สุด
* ไม่ปิดแท็บแรกที่โปรแกรมเปิดขึ้นมา
* ทดสอบทีละขั้นก่อนรันงานจริง
